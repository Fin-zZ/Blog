# 内联元素

# line-height

以字母x为基准，x的下边缘即为 baseline基线， x的中点近似为 middle的值。所以不同字体位置可能不一。

内敛元素默认都是基线baseline对齐的。

line-height: 非替换内联元素的行高 替换元素指可通过属性更改值变化内容的（img的src input的type select这种） 所以文本这种完全由line-height决定行高度了

以上的行距line-height分别为2倍、1倍、0.5倍。 可以看出来2倍的时候，其实行间距就正好等于1倍的font-size， 原因：行距 = 行高 - em-box => 行距 = line-height - font-size em-box是一个相对于font-size的尺寸，1em = 当前元素font-size的设定值

上行距 = Math.floor(行距 / 2) 下行距 = Math.ceil(行距 / 2)

所以，设定完line-height后，其实文字是自动默认（近似）居中的（上下行距）。 其次原因就是不同字体 可能略有差别啦。

并且对多行文字每一行控制是一样的。

接着是替换元素还有块状元素 和 line-height的关系 对于替换元素，他们本质还是内联元素，所以在h5的文档模式下， 前面还是有一个宽度为0的空白幽灵节点存在，它的特性就和普通文字一样， 导致容器的最小高度会是line-height的值；因为非替换元素不受line-height控制， 以及vertical-align会影响。

日常的开发里，icon文字共存非常普遍，但这俩一个是替换元素（img），一个是非替换元素（文字） 放在同一行无法对齐。

对于block级元素，line-height无法控制其高度，但会通过影响里面元素的高度撑开block的高度。

对于多行文本 以及 图片类(替换元素) 的居中设置需要借助vertical-align:middle

.box{
    line-height: 180px;
    background-color:cornflowerblue;
}
.content{
    display: inline-block;
    width: 150px;
    font-size: 16px;
    line-height: 24px;
    vertical-align: middle;
    background-color:greenyellow;
}
对于block级元素，line-height无法控制其高度，但会通过影响里面元素的高度撑开block的高度。

原理： 多行文字通常也在一个标签内，再将content设为inline-block,就能保证内部使用内部的line-height， 外部整体使用一个line-height，外部能使用的原因是存在空白幽灵节点，它使得外部line-height有了作用对象， 由此撑开了最小高度，多行和替换元素居中需要借助 vertical-align:middle,实现近似居中。

# vertical-align

默认值：baseline，也就是x的下边缘，内联元素都是默认沿x下边缘对齐的。 但是对于img这种替换元素，用元素本身的下边缘作为基线。所以一般而言， 图片下边缘 与 x的下边缘对齐。

vertical-align的百分比%是基于line-height计算的 其次是只能作用于内联元素和 display:table-cell的元素

还有一个特殊情况，display: inline-block的元素的vertical-align， 如果该标签里没有内联元素，或者overflow不是visible，那么基线就是其margin的边缘。 如果里面有元素，那就是最后一行的内联元素的基线。

.dib-baseline {
    display: inline-block;
    width: 150px;
    height: 150px;
    border: 1px solid #cad5eb;
    background-color: #f0f3f9; 
}

<div>
  <div class="dib-baseline">x</div>
  <div class="dib-baseline"></div>
